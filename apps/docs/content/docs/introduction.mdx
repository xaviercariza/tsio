# Introduction

`tsio` is a contract-first TypeScript library for defining type-safe realtime protocols.

It lets you describe the messages your application understands once, then reuse that contract to type:

- server action handlers
- generated client actions
- generated client events
- server-to-client event emission
- middleware context refinement
- Socket.IO and `ws` transport adapters

## The problem

Realtime code often starts with strings and `any` payloads:

```ts
socket.emit('send-message', { text })
socket.on('message-received', payload => {
  // What is payload?
})
```

That works until event names drift, payloads change, or authentication context is nullable in every handler.

`tsio` replaces that with a shared contract:

```ts
const api = contract({
  chat: {
    sendMessage: {
      type: 'action',
      input: z.object({ text: z.string() }),
      response: MessageSchema,
    },
    onMessage: {
      type: 'event',
      data: MessageSchema,
    },
  },
})
```

From that contract, the server and client both receive inferred types.

## When to use it

Use `tsio` when your app needs typed live communication:

- chat and presence
- multiplayer rooms
- collaborative interfaces
- dashboards with live updates
- admin consoles with realtime feedback
- internal tools using Socket.IO or `ws`

For plain request/response HTTP APIs, `tsio` is usually not the right tool. Use an HTTP-focused API approach instead.
