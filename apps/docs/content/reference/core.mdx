# Core API Reference

This page summarizes the public API exported from `@tsio/core`.

## `contract`

Defines a typed protocol contract and preserves literal and schema types.

```ts
const api = contract({
  chat: {
    sendMessage: {
      type: 'action',
      input: z.object({ text: z.string() }),
      response: MessageSchema,
    },
    onMessage: {
      type: 'event',
      data: MessageSchema,
    },
  },
})
```

## `createServer`

Creates a typed server builder for a contract.

```ts
const tsio = createServer.context<Context>().create(api)
```

The returned object contains:

- `router`
- `middleware`
- `attach`

## `router.create`

Creates a server router from action builders.

```ts
const router = tsio.router.create(a => ({
  chat: {
    sendMessage: a.chat.sendMessage.handle(handler),
  },
}))
```

Nested router factories are also available:

```ts
const chatRouter = tsio.router.chat.create(a => ({
  sendMessage: a.sendMessage.handle(handler),
}))
```

## `middleware`

Creates middleware that can refine the handler context.

```ts
const requireUser = tsio.middleware(async ({ ctx, next }) => {
  if (!ctx.user) return { ok: false, error: new Error('Unauthorized') }
  return next({ ctx: { user: ctx.user } })
})
```

Middleware can be used directly or piped.

## `attach`

Attaches a server router to a transport adapter.

```ts
tsio.attach({ router, adapter, createContext })
```

## `createClient`

Creates a typed client from a contract and client adapter.

```ts
const client = createClient(api, adapter)
```

The client exposes `client.actions` and `client.events`.

## `mergeContracts`

Merges multiple contract objects.

```ts
const api = mergeContracts(authApi, chatApi, presenceApi)
```

Use this when splitting a large protocol across files.
